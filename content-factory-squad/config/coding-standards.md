# Coding Standards — content-factory-squad

## Linguagem
- Markdown para conteúdo longo (artigos, blog posts, whitepapers)
- YAML para configurações e calendários editoriais
- JSON para dados estruturados (metadados, posts sociais, briefs)
- Templates Markdown para formatos reutilizáveis

## Convenções de Nomes
- Variáveis e funções: camelCase (`contentCalendar`, `writeLongForm`)
- Constantes: UPPER_SNAKE_CASE (`MAX_HASHTAGS`, `DEFAULT_KEYWORDS`)
- Arquivos: kebab-case (`plan-content-calendar.md`, `long-form-content.md`)
- IDs de conteúdo: `CFS-YYYY-NNNN` (`CFS-2026-0001`)
- Calendários: kebab-case com data (`content-calendar-2026-03.md`)

## SEO Standards
- Title Tag: < 60 caracteres, keyword principal no início
- Meta Description: < 160 caracteres, CTA incluído
- Heading Structure: H1 única, H2-H4 hierárquicos com keywords
- Keyword Density: 1-2% no corpo do texto, sem keyword stuffing
- Alt Text: descritivo para todas as imagens, incluir keyword quando natural
- URL Slug: kebab-case, curto, com keyword principal

## Conteúdo
- UTF-8 para todos os arquivos de conteúdo
- Parágrafos curtos (3-5 linhas) para legibilidade web
- Bullet points para listas de benefícios ou features
- CTAs claros e alinhados à fase do funil
- Fontes e referências citadas quando aplicável

## Social Media
- Hashtags: 5-15 por post Instagram, 3-5 LinkedIn, 2-3 Twitter
- Hooks: primeiras 2 linhas do caption devem capturar atenção
- Emojis: usar com moderação, alinhados à brand voice
- Links: UTM parameters em todos os links trackáveis
- Menções: @mencionar parceiros e fontes relevantes

## Imagens
- Dimensões: sempre corretas para a plataforma alvo
- Formato: PNG para gráficos/texto, JPEG para fotos
- Tamanho: < 5MB para uploads em redes sociais
- Alt text: obrigatório para acessibilidade
- Brand guidelines: cores, fontes e estilo consistentes

## Versionamento
- Conteúdo versionado por data de criação
- Revisões marcadas com sufixo `-v2`, `-v3`
- Changelog em arquivo separado para whitepapers e e-books
